# Doubt-Driven Development

Confident ≠ correct. Materialize a fresh reviewer biased to disprove.

When: Non-trivial decisions — branching logic, module boundaries, unverifiable invariants, irreversible operations.

Process: 1.CLAIM (2-3 lines) 2.EXTRACT (artifact+contract, no reasoning) 3.DOUBT (adversarial prompt) 4.RECONCILE (classify) 5.STOP (trivial findings, 3 cycles, or user override).
