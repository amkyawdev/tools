# Git Workflow and Versioning

Git = safety net. Commits = save points. History = documentation.

Core: Trunk-based (1-3 day branches), commit early/often, atomic commits, descriptive messages (feat/fix/refactor/test/docs/chore), separate concerns, ~100 lines per change.
Branch: feature/, fix/, chore/, refactor/. Change summaries: what, didn't touch, concerns.
