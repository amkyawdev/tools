# Planning and Task Breakdown

Decompose work into small, verifiable tasks with explicit acceptance criteria.

## Process
1. Enter plan mode (read-only — no code)
2. Identify dependency graph (DB → API → Frontend)
3. Slice vertically (one complete feature path at a time)
4. Write tasks with: description, acceptance criteria, verification, dependencies, files
5. Order and add checkpoints every 2-3 tasks

## Task Sizes: XS (1 file) / S (1-2 files) / M (3-5 files) / L (5-8 files) / XL (break down)

## Parallelization: Independent slices can run in parallel. Sequential for migrations/shared state.
