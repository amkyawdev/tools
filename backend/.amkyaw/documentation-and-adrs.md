# Documentation and ADRs

Document decisions (why), not just code (what).

ADR: Status, Date, Context, Decision, Alternatives, Consequences. Store in docs/decisions/.

Comment rules: explain why (not what), document gotchas. Never comment obvious code, leave TODOs, or leave commented-out code.

README: Quick Start, Commands, Architecture, Contributing.
Agent docs: CLAUDE.md, specs, ADRs, inline gotchas.
